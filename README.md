## Патерн проєктування: Спостерігач (Observer)

У проєкті реалізовано патерн **Спостерігач** для системи сповіщень про зміну доступності номерів.

### Як це працює

| Роль | Компонент |
|------|-----------|
| **Subject (Суб'єкт)** | Модель `Room` — при зміні `is_available` генерує подію через Django signals |
| **Observer (Спостерігач)** | Модель `Subscription` — представляє підписника (користувача на конкретний номер) |
| **Notification (Сповіщення)** | Модель `NotificationLog` — лог усіх надісланих сповіщень |
| **Signal handlers** | `hotel/signals.py` — `pre_save` + `post_save` на `Room` |

### Функціонал для користувача

1. **Підписка** — на сторінці номера (`/rooms/<id>/`) натисніть «Підписатися на сповіщення» (потрібен вхід через `/login/`, нові користувачі можуть зареєструватися через `/register/`)
2. **Відписка** — там само кнопка «Відписатися від сповіщень» (доступно для автентифікованих користувачів)
3. **Мої підписки** — сторінка `/subscriptions/` показує всі підписки та логи сповіщень (доступна після входу в систему)

### Адмін-панель

* **Підписки** → `admin/hotel/subscription/` — хто підписаний і на який номер
* **Логи сповіщень** → `admin/hotel/notificationlog/` — повний журнал з фільтрами по даті, кімнаті, статусу

### Тестування Observer

```bash
python manage.py test hotel.tests.ObserverSubscriptionTest
```

## Як запустити проєкт локально (без Docker)

Цей спосіб підійде для розробки та написання коду.

### 1. Клонування репозиторію
Відкрийте термінал та виконайте команду:
```bash
git clone https://github.com/AndreyShyshkin/Python-Hotel
cd Python-Hotel
```

### 2. Встановлення залежностей
```bash
pip install -r requirements.txt
```

### 3. Налаштування бази даних

Застосуйте міграції для створення необхідних таблиць у базі даних:

```bash
python manage.py migrate
```

### 4. Створення суперкористувача (для доступу в Admin-panel)

Щоб керувати номерами, категоріями та іншими даними, створіть адміністратора:

```bash
python manage.py createsuperuser
```
(Дотримуйтесь інструкцій у терміналі: введіть логін, email та пароль)


### 5. Генерація тестових даних (Seeding)

Щоб не створювати категорії, зручності та номери вручну через адмінку, ви можете скористатися нашою кастомною командою, яка автоматично наповнить базу даних готовими номерами з фотографіями:

```bash
python manage.py seed_rooms          # створити 5 номерів (пропустити, якщо вже є)
python manage.py seed_rooms --reset  # спочатку видалити всі старі дані, потім пересіяти наново
```

### 6. Запуск сервера розробки

```bash
python manage.py runserver
```

Після цього сайт буде доступний за адресою: http://127.0.0.1:8000/ </br>
Адмін-панель доступна за адресою: http://127.0.0.1:8000/admin/

## Запуск за допомогою Docker

Це найшвидший та найзручніший спосіб розгорнути проєкт на будь-якому комп'ютері без необхідності локально встановлювати Python та налаштовувати віртуальне середовище.

### Передумови
Переконайтеся, що у вас встановлено [Docker Desktop](https://www.docker.com/products/docker-desktop) (або Docker Engine + Docker Compose).

### 1. Клонування репозиторію
Якщо ви ще не клонували проєкт, виконайте:
```bash
git clone https://github.com/AndreyShyshkin/Python-Hotel
cd Python-Hotel
```

### 2. Запуск контейнерів

У кореневій папці проєкту (де знаходиться файл docker-compose.yml) виконайте команду:

```bash
docker-compose up --build
```

(Щоб запустити сервер у фоновому режимі та звільнити термінал, використовуйте прапорець -d: docker-compose up -d --build)

### 3. Автоматичне наповнення бази даних та доступ до адмінки

В рамках нашого пайплайну при старті Docker-контейнерів автоматично виконуються міграції, генеруються тестові дані (seeding) та створюється суперкористувач. Вам не потрібно виконувати жодних додаткових налаштувань вручну.

Після запуску `docker-compose up` у базі автоматично з'являться:
1. **5 тестових номерів** (разом із категоріями, зручностями та підключеними фотографіями) для миттєвої демонстрації роботи сайту.
2. **Суперкористувач** для доступу в панель керування.

### 4. Доступ до сайту
Після успішної збірки та запуску веб-застосунок буде доступний у браузері за адресою: **http://localhost:8000**

### 5. Доступ до адмінки
1. Відкрийте в браузері: **http://localhost:8000/admin**
2. **Логін:** `admin`
3. **Пароль:** `Admin_12345!`

*Примітка: Ці креденшиали задані через environment variables у файлі docker-compose.yml виключно для локальної розробки та тестування.*

### 6. Зупинка контейнерів

Щоб зупинити роботу сервера (якщо він був запущений через -d), виконайте:

```bash
docker-compose down
```
